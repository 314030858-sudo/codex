# Windows 安装和试用说明

这份说明给非技术试用人员使用。当前安装包是无签名测试版，只用于内部试用，不是正式发布版本。

## 一、怎么下载安装包

1. 打开本项目的 GitHub 页面。
2. 进入本次 PR 页面。
3. 打开 PR 里的自动构建记录，也就是 Windows installer 构建。
4. 在页面底部找到 artifact，名称是 `yinghe-ai-windows-bundle`。
5. 下载这个 artifact，解压后会看到一个安装程序，文件名类似 `影核 AI_0.1.0_x64-setup.exe`。

如果看不到 artifact，说明自动构建还没完成，或者构建失败，需要先让开发同学处理。

## 二、怎么安装

1. 双击 `影核 AI_0.1.0_x64-setup.exe`。
2. 按安装向导提示继续。
3. 安装完成后，在开始菜单或桌面里找到“影核 AI”。

## 三、Windows 安全提示怎么处理

当前是无签名测试版，Windows 可能会提示“Windows 已保护你的电脑”。

处理方式：

1. 点击“更多信息”。
2. 确认应用名称是“影核 AI”。
3. 点击“仍要运行”。

注意：这只是内部测试阶段的临时处理方式。正式给大量真实用户使用前，建议补正式代码签名证书。

## 四、怎么打开软件

安装后打开“影核 AI”。正常情况下，会看到本地媒体库页面和“导入本地文件夹”的入口。

## 五、怎么导入本地照片

1. 准备一个测试文件夹，里面放少量照片或视频。
2. 建议先用复制出来的测试照片，不要直接拿唯一原图做第一次验证。
3. 在软件里点击“导入本地文件夹”。
4. 选择刚才准备的测试文件夹。
5. 等待扫描完成。

当前导入的含义是：软件记录本地文件路径和基础信息，不会把照片复制到云端。

## 六、怎么确认 EXIF 功能生效

导入带相机信息的照片后，在媒体卡片里查看：

- 是否显示拍摄时间。
- 是否显示相机品牌或型号。
- 搜索框里输入相机型号、拍摄年份或文件名，是否能找到对应照片。

如果某张照片没有显示相机信息，可能是照片本身没有 EXIF，或者 EXIF 被微信、截图工具、修图软件清理过。

## 七、怎么确认没有上传照片

当前版本没有登录、没有云同步、没有外部云服务调用入口。

试用时可以这样确认：

1. 断开网络后打开软件。
2. 导入测试文件夹。
3. 确认仍然可以看到本地照片记录和 EXIF 信息。

如果断网后仍能导入和查看，说明这个流程不依赖上传。

## 八、怎么确认没有删除原图

导入前后检查测试文件夹：

1. 原照片文件还在原来的文件夹里。
2. 文件名没有变化。
3. 文件数量没有减少。
4. 文件位置没有变化。

当前版本只扫描和记录本地文件信息，不做删除、移动、重命名原图的动作。

## 九、本地打包命令给开发同学

执行目录：

```powershell
C:\Users\07\Documents\Codex\2026-05-28\github-issue-4-https-github-com\apps\desktop
```

推荐命令：

```powershell
npm.cmd run bundle:windows
```

等价 Tauri 命令：

```powershell
npm.cmd run tauri -- build --bundles nsis
```

打包结果：

```text
apps/desktop/src-tauri/target/release/bundle/nsis/
```

安装包文件名类似：

```text
影核 AI_0.1.0_x64-setup.exe
```

说明：如果直接执行 `npm run ...` 被 Windows PowerShell 拦截，可以使用 `npm.cmd run ...`。
