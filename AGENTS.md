# AGENTS.md

这是“影核 AI”项目给 Codex / AI Agent 的工作规则。

## 项目总目标

开发一个本地优先的超级智能照片、视频整理与 AI 修图软件。

最终产品应成为：

- 本地个人相册。
- 自动整理照片和视频的 AI 系统。
- 自然语言找图工具。
- AI 修图软件。
- 视频素材管理工具。
- 企业视觉资产库。

## 当前阶段策略

不要一开始做复杂大系统。必须按阶段推进：

1. 项目文档。
2. 桌面空壳。
3. 本地媒体导入。
4. 缩略图系统。
5. 智能相册第一版。
6. 搜索第一版。
7. AI Worker 第一版。
8. 自然语言搜图。
9. 基础非破坏式修图。
10. 人像精修。
11. 视频整理。

## 技术选型

- 桌面端：Tauri + React + TypeScript。
- 本地核心：Rust。
- AI Worker：Python。
- 数据库：SQLite。
- 图片处理：先用稳定简单方案，后续优化到 libvips / OpenCV。
- 视频处理：后续接 FFmpeg / ffprobe。
- AI 搜索：后续接 CLIP 或兼容本地模型。

## 隐私原则

1. 用户照片、视频、人脸、企业素材默认不上传。
2. 默认本地处理。
3. 后续任何云能力必须是可选项。
4. 不得默认采集用户媒体内容。
5. 人脸数据必须可删除。

## 修图原则

1. 原图永远不修改。
2. 所有修图参数保存在 edit recipe 中。
3. 支持随时回退。
4. 批量修图必须可预览、可撤销。

## 禁止事项

- 不要一次性实现多个阶段。
- 不要在阶段 2 接入 SQLite、AI、扫描或修图。
- 不要默认上传任何用户媒体。
- 不要复制竞品品牌、界面、模型、素材或专有工作流。
- 不要为了炫技写难以维护的代码。

## 代码要求

- 组件命名清晰。
- 模块职责单一。
- 保持可测试性。
- 重要变更更新文档。
- 每次完成任务后说明：
  1. 修改了哪些文件。
  2. 怎么运行。
  3. 怎么测试。
  4. 没有实现哪些功能。

## 阶段 2 验收标准

- 桌面窗口能启动。
- 显示“影核 AI”。
- 有左侧导航栏、顶部搜索框、导入按钮、空状态、底部状态栏。
- 不包含真实导入、数据库、AI、修图或视频处理。
